Попрактиковаться в написании SQL-запросов:

Простой поиск и выборка отдельных полей по условию
Составные условия: AND, OR, IS NULL / IS NOT NULL
Выборка уникальных значений: использование DISTINCT
Сортировка результатов поиска: ORDER BY
Объединение данных из нескольких таблиц: использование JOIN
Группировка результатов запроса с помощью GROUP BY и агрегатные функции: COUNT, MAX, MIN, SUM
Объединение результатов нескольких запросов в один: UNION
* Сложные условия: использование HAVING

Интернет-магазин мебели:

Основные сущности в базе данных - поставщик, покупатель, товар, заказ
Есть несколько поставщиков, данные о них хранятся в таблице producers
Данные о покупателях хранятся в таблице customers
Каждый покупатель может иметь один или несколько адресов для доставки (например, дом и работа).
Адреса хранятся в таблице shipping_addresses. При создании заказа покупатель выбирает один из адресов
Данные о товарах хранятся в таблице items.
У каждого товара указывается его поставщик и доступное на данный момент количество единиц в магазине
Данные о заказах хранятся в таблицах orders и orders_content.
Каждому заказу соответствует одна запись в таблице orders, где указывается покупатель и адрес доставки,
и одна или несколько записей в orders_content,
каждая из которых содержит id товара и сколько его единиц заказано.

Задания:
(prod - items: 1-N, items - orders: N-N (through orders_content), customers-orders: 1-N,
customers - shipping address: 1-N ((can be 2+ addresses)))
user_id      int references _user (id) --> 1:N ((user --> accountS)) (can't see it in initial project)

1. Вывести имена и фамилии всех покупателей
2. Для каждого товара вывести доступный остаток, отсортировать по убыванию остатка
3. Отобразить все товары, для которых заполнено описание
4. Отобразить список id покупателей, для которых существует хотя бы один заказ
5. Вывести информацию о всех диванах (где название или описание начинается со слова "диван")
6. Вывести все товары поставщика "1000 диванов"
7. Выбрать данные обо всех заказах (id заказа, название товара, заказанное количество,
имя и фамилия покупателя, город и улица доставки)
8. Для каждого покупателя вывести данные обо всех его адресах
9. Вывести список всех городов, из которых присутствует либо покупатель, либо поставщик
10. Добавить нового покупателя (без заказов)
11. Для каждого покупателя вывести количество заказов
12. Посчитать, сколько заказали экземпляров каждого товара
13. Вывести имена и фамилии всех покупателей, живущих в Москве (у которых там хотя бы один адрес)
14. Найти, сколько было заказов на каждый адрес
15. * Для каждого поставщика отобразить общее проданное количество товаров и доступный остаток
